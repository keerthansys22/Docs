\hypertarget{index_intro_sec}{}\doxysection{\texorpdfstring{Introduction}{Introduction}}\label{index_intro_sec}
This documentation contains the Industrial Valve Island (Arrow3 product) linux application contents. Here we have Main bus node, IO modules and Valves connected.

{\bfseries{IO modules}} can contain following modules\+:-\/
\begin{DoxyItemize}
\item DIO
\item DO
\item DI
\item AIO
\item AO
\item AI
\end{DoxyItemize}

{\bfseries{Valves}} can contain the following\+:-\/
\begin{DoxyItemize}
\item {\bfseries{Pressure zone}}\+:-\/ ISEM diagnostics valve, No of stations(\+Standard valves), No of diagnostics valves, Proportional station valves(\+Internal prop)
\item {\bfseries{External pressure zone}}\+:-\/ Proportional module valves(\+External prop)
\end{DoxyItemize}\hypertarget{index_install_sec}{}\doxysection{\texorpdfstring{Overview}{Overview}}\label{index_install_sec}
The LCD GUI application operates on a Linux core, communicating with a data server for data exchange. The application includes five threads, each dedicated to specific tasks to ensure modularity, scalability, and performance. \hypertarget{index_step1}{}\doxysubsection{\texorpdfstring{{\bfseries{Key Features}}}{{\bfseries{Key Features}}}}\label{index_step1}
\hypertarget{index_step2}{}\doxysubsection{\texorpdfstring{{\bfseries{Read Operations}}\+:}{{\bfseries{Read Operations}}\+:}}\label{index_step2}

\begin{DoxyItemize}
\item Fetch MAC address
\item Module details (Part number, Hardware and Firmware revision)
\item PIN details
\item IP address details (default values)
\item module status (device type, status value, scrolling status and suggestion details)
\item Poll status details from the data server on configurable interval. 
\end{DoxyItemize}\hypertarget{index_step3}{}\doxysubsection{\texorpdfstring{{\bfseries{Write Operations}}\+:}{{\bfseries{Write Operations}}\+:}}\label{index_step3}

\begin{DoxyItemize}
\item Send user-\/entered IP configuration details to the data server. 
\end{DoxyItemize}\hypertarget{index_step4}{}\doxysubsection{\texorpdfstring{{\bfseries{Threads and Responsibilities}}\+:}{{\bfseries{Threads and Responsibilities}}\+:}}\label{index_step4}

\begin{DoxyItemize}
\item {\bfseries{Thread 1\+: LVGL State Machine}}
\begin{DoxyItemize}
\item Manages the lifecycle and states of the LVGL GUI framework.
\end{DoxyItemize}
\item {\bfseries{Thread 2\+: LVGL GUI}}
\begin{DoxyItemize}
\item Handles GUI rendering and user interface interactions on the display.
\end{DoxyItemize}
\item {\bfseries{Thread 3\+: Data processing polling}}
\begin{DoxyItemize}
\item Continuously polls the status details for all the modules in a hierarchy.
\end{DoxyItemize}
\item {\bfseries{Thread 4\+: Poll for Screen Alerts}}
\begin{DoxyItemize}
\item Continuously polls for any screen pop-\/ups indicating warning or fault statuses.
\end{DoxyItemize}
\item {\bfseries{Thread 5\+: Keypress Check}}
\begin{DoxyItemize}
\item Monitors keypress events for navigation and user actions. 
\end{DoxyItemize}
